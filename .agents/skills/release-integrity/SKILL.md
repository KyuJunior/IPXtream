# release-integrity

This skill guarantees the consistency, correctness, and automation compliance of IPXtream releases, preventing pipeline failures and ensuring the in-app update checks work seamlessly.

## Core Guidelines

### 1. Version Bumping Rules
* **Mandatory Version Increment**: After any change or feature implementation, you must increment the version number by exactly `+0.0.1` (e.g., `2.0.90` to `2.0.91`) before building the release package.
* **Synchronized Target Files**: Version updates must be applied synchronously across all configuration files:
  1. `IPXtream/IPXtream.csproj`: The `<Version>` tag.
  2. `IPXtream_Installer.iss`: The `#define MyAppVersion` and publishing output comment paths.
  3. `CONTEXT.md`: The development history, release details, and active version numbers.
* **Automated Bump Execution**: Rely on the script [swalalala.py](file:///c:/Myapps/ipxtream/swalalala.py) to automatically handle version bumping, dotnet publishing, installer compilation via Inno Setup (ISCC), patch notes generation, and GitHub release creation.

### 2. Pre-Release Verification & Sanity Checks
* **Dotnet Build Sanity**: Before invoking `swalalala.py`, ensure the code compiles without errors:
  ```powershell
  dotnet build IPXtream\IPXtream.csproj
  ```
* **Git Clean State Check**: Avoid executing the automated release workflow if there are unstaged changes that should not be released, or unresolved merge conflicts.
* **Environment Setup**: Ensure your shell environment does not have a stale/dummy `GITHUB_TOKEN` environment variable that overrides `gh` CLI keyring credentials. Clear it if needed:
  ```powershell
  $env:GITHUB_TOKEN = $null; $env:GH_TOKEN = $null
  ```

### 3. Installer Asset & In-App Update Naming Constraints
For the in-app auto-update checker to parse releases correctly, comply with the following strict naming conventions:
* **Setup File Pattern**: The setup file generated by Inno Setup must follow the format `IPXtream_Setup_v{Version}.exe` (e.g., `IPXtream_Setup_v2.0.91.exe`).
* **GitHub Release Upload**: The setup installer must be uploaded using `gh release create` matching the tag name exactly (e.g., tag `v2.0.91` matches version `2.0.91`).
* **Fallback Endpoint Compatibility**: The fallback download mechanism constructs download URLs in the format `https://github.com/KyuJunior/IPXtream/releases/download/{tagName}/IPXtream_Setup_{tagName}.exe`. Ensure the asset uploaded is named matching `IPXtream_Setup_{tagName}.exe` (where `{tagName}` is `v{Version}`).
* **Commit History Tracking**: After running `swalalala.py`, commit and push all updated version files to the main branch immediately.
